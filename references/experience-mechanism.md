# 经验积累机制

> 本文件由 paper-workflow 主流程引用。每个 Phase 结束、有新发现需要沉淀时，按本文件流程与模板写入。

每次使用 skill 调参过程中产生的新发现，Phase 结束之后展示给用户：

> 本次发现：
> - [发现 1]
> - [发现 2]
>
> 是否写入经验区？

用户确认后才写入。写入时分两类：

| 类型 | 存储位置 | 内容 |
|------|---------|------|
| **通用规律** | 写入主 SKILL.md 的「积累的经验」区 | 与具体项目/电机参数无关的跨项目规律（如 "ESO 误差符号必须与控制律一致"） |
| **项目特定记录** | 写入项目 `docs/superpowers/experience.md` | 与特定电机参数、特定论文、特定工况绑定的经验（如 "电机 A 的 Lz=4.2mH 时 STSM 增益需设 lam1≥20000"），主 SKILL.md 中加一行引用链接 |

## 经验条目模板（通用规律）

每条经验必须包含以下元数据：

```markdown
### 经验 #N: [一句话标题]

| 字段 | 内容 |
|------|------|
| 适用算法 | STSM / LESO / ADRC / SMC / ... |
| 适用条件 | [什么情况下这条经验生效，用于 agent 自动匹配] |
| 编辑日期 | YYYY-MM-DD |
| 触发阶段 | Phase 1 / 2 / 3 / 4 |

**经验内容：**
[详细描述发现的规律、原因和应对方法]
```

> **Agent 匹配规则：** 逐条比对当前任务的算法类型、工况与每条经验的 `适用条件` 字段。条件匹配的经验强制执行；条件不匹配的跳过。

多条经验按编辑日期倒序排列。
